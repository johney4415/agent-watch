# Contributing

Thanks for helping improve Agent Watch.

1. Open an issue for substantial behavior or API changes.
2. Create a focused branch from `main`.
3. Run `swift test` before opening a pull request.
4. Keep the app local-first and avoid adding dependencies without a clear benefit.

Bug reports should include the macOS version, terminal app, agent CLI and version, expected behavior, and a redacted example event when relevant. Never include prompts, credentials, or private repository paths in public issues.
